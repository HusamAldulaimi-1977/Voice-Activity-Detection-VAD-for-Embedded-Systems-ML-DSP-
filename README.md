The VAD system was evaluated under different experimental conditions, including varying signal-to-noise ratio (SNR) levels, different MSE percentile thresholds, and different window sizes for majority voting smoothing used in decision making.
The VAD application was used to generate CSV files containing frame-level decisions, including time, frame index, and binary classification (speech or noise). These outputs were then processed in MATLAB, where manually generated ground truth labels were used to compute confusion matrices and evaluate system performance.
Ground truth labels were manually created in MATLAB using an interactive labeling tool. The predicted labels were aligned with the ground truth on a frame-by-frame basis to compute the confusion matrix components: true positives (TP), true negatives (TN), false positives (FP), and false negatives (FN).
The input audio file is loaded into MATLAB. The CSV file generated by the VAD application is also loaded. Manual labeling is then performed by pressing the start button and using the two buttons (noise and speech+noise) to generate the ground truth for the audio file. After labeling is completed, the “Evaluate Now” button extracts the decision column from the CSV file, compares it with the ground truth on a frame-by-frame basis
